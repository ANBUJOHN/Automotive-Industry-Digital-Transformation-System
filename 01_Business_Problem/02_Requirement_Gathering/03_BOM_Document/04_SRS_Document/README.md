# Software Requirement Specification (SRS)

## 1. Introduction

### 1.1 Purpose
The purpose of this document is to define the software requirements for the Automotive Industry Digital Transformation System. It provides a detailed description of system functionalities and serves as a reference for development and validation.

### 1.2 Scope
The system is designed to automate and manage business processes in the automotive industry, including user management, part tracking, parameter monitoring, and dashboard reporting.


## 2. Overall Description

### 2.2 Product Functions
- User Authentication (Login/Logout)  
- User Management (User Master)  
- Part Management (Part Master)  
- Dashboard Monitoring  
- Email Notification System  


## 3. Functional Requirements

### 3.1 Login Module
- Users must log in using valid credentials  
- System should validate username and password  
- Invalid login should show error message  

### 3.2 User Master
- Add, edit, delete users  
- Assign roles and permissions  

### 3.3 Part Master
- Add and manage part details  
- Track part information  


### 3.4 Dashboard
- Display real-time data  
- Show reports and analytics  

### 3.5 Email Module
- Send automated notifications  
- Alert users based on system events  

---

## 4. Non-Functional Requirements

### 4.1 Performance
- System should respond within acceptable time limits  

### 4.2 Security
- Role-based access control  
- Secure login authentication  

### 4.3 Usability
- User-friendly interface  
- Easy navigation  

### 4.4 Reliability
- System should operate without failure  


## 5. System Design

### 5.1 UI/UX Design
The system includes user-friendly screens such as:
- Login Page  
- Dashboard  
- Master Management Screens
